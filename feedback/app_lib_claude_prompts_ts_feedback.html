app/lib/claude/prompts.ts Feedback

Target file: app/lib/claude/prompts.ts Feedback type: Prompt boundary and data-contract architecture

Elephant In The Room

This file currently assembles large prompt contexts where key facts, preferences, constraints, and sources are mostly rendered as text. That makes the LLM boundary too loose for the quality bar.

Major Feedback

Suggested Implementation Direction

  • Add prompt builders that accept CandidateExperience[], ScheduleConstraint[], and ValidationFailure[].
  • Serialize compact structured blocks rather than long prose blocks wherever possible.
  • Make every LLM response preserve IDs from its input so downstream code can validate it.

Principal Engineer Implementation Notes

Prompt builders should serialize contracts that code can validate. Keep prompts expressive, but make IDs and constraints non-negotiable.

Implementation-Ready Checklist

Acceptance Criteria

  • Downstream code can validate every LLM output against known IDs.
  • The LLM can still write delightful copy, but cannot silently swap or invent primary entities.
  • Reviewer prompts become repair prompts over known validation failures.

Needs Product Decision

  • Decide how much raw source text belongs in prompts after candidate extraction. Recommendation: pass compact extracted facts and source refs, not full raw snippets, after the inventory stage.
