Target file: app/scripts/eval-grounding.ts Feedback type: Grounding eval architecture

Elephant In The Room

The grounding eval checks whether expected names appear, but it does not yet measure whether the grounding layer is strong enough to become the system of record for the board and planner.

Major Feedback

Suggested Implementation Direction

  • Add checks for required fact types in golden specs.
  • Score source-backed facts separately from name presence.
  • Fail or warn when required logistics facts are missing even if the experience itself is found.

Principal Engineer Implementation Notes

Make grounding eval judge whether sources can support downstream planning, not just whether names appear.

Implementation-Ready Checklist

Acceptance Criteria

  • Grounding can fail even when all required names are present if planning-critical facts are missing.
  • The output tells generation exactly which fact categories need better sourcing.
  • The eval distinguishes weak evidence from absent evidence.

Needs Product Decision

  • Decide which fact categories are required for v1 schedule-readiness. Recommendation: duration, access mode, coordinates/place ID, family suitability, and at least one source ref.
