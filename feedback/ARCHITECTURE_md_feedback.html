ARCHITECTURE.md Feedback

Target file: ARCHITECTURE.md Feedback type: System architecture

Elephant In The Room

The architecture still describes a mostly LLM-first generation system. The missing architectural box is a normalized travel intelligence layer that sits between search/Places/weather and the LLM writers/planners.

Major Feedback

Suggested Architecture Shape

Search + Places + Weather + curated rules
  -> Verified candidate inventory
  -> Fact and constraint extraction
  -> Board selection and card writing
  -> Board validation gates
  -> Deterministic itinerary anchors and constraints
  -> LLM itinerary composition
  -> Deterministic itinerary validation

The LLM remains important, but its job becomes synthesis and explanation over reliable data instead of being the reliability mechanism itself.

Principal Engineer Implementation Notes

Change the architecture diagram so the LLM is no longer the system of record. Add the missing intelligence layer explicitly.

Implementation-Ready Checklist

Acceptance Criteria

  • A new developer can tell which stage owns truth, which stage owns prose, and which stage owns scheduling.
  • The architecture describes what happens when grounding, Places, validation, or review fails.
  • The diagram makes hard constraints visible before itinerary planning.

Needs Product Decision

  • Pick a name for the canonical layer: VerifiedDestinationInventory, DestinationGraph, or CandidateInventory. Recommendation: VerifiedDestinationInventory because it is precise and not over-engineered.
